## Description

[SRP] Single Responsibility Principle:
O Princípio da responsabildade Única Uma classe deve ser responsável por apenas uma parte específica da funcionalidade do software e não deve assumir responsabilidades adicionais.

[OCP] - Open-Closed Principle:
O Princípio Aberto-Fechado afirma que uma classe, módulo ou função deve estar aberta para extensão, mas fechada para modificação. Em outras palavras, o comportamento de uma entidade de software deve poder ser estendido sem alterar seu código-fonte original.

[LSP] Liskov Substitution Principle:
O Princípio da substituição de Liskov implica em que objetos podem ser substituídos por seus subtipos sem que isso afete a execução correta do programa.

[ISP] Interface Segregation Principle:
O Princípio da Segregação da Interface afirma que os clientes não devem ser forçados a depender de interfaces que eles não utilizam. Em outras palavras, em vez de ter uma única interface ampla e abrangente, é melhor dividir as interfaces em grupos menores e mais específicos, de modo que os clientes só precisem conhecer e implementar os métodos que realmente utilizam.

[DIP] Dependency Inversion Principle:
O princípio da inversão de dependência diz que devemos depender de abstrações (interfaces) ao invés de implementações (classes concretas) a fim de ter um menor acoplamento entre as camadas do sistema.

## Installation

```bash
$ yarn install
```

## Running the app

```bash
# development
$ yarn run start

# watch mode
$ yarn run start:dev

# production mode
$ yarn run start:prod
```

## Test

```bash
# unit tests
$ yarn run test

# e2e tests
$ yarn run test:e2e

# test coverage
$ yarn run test:cov
```
